Chapter Two

Three tools, one stack.

You don't need to learn everything. You need three things: a place for code, a place for the app, and a place for the data.

i.

GitHub

Where the code lives

A version-controlled home for your project. Every change is saved, every version is recoverable, and your work is backed up to the cloud.

"It's Google Drive — but for code, with infinite undo."
ii.

Vercel

Where the app runs

Connects to your GitHub repo and turns your code into a live website on a real URL. Push a change, and the live site updates in seconds.

"It's the printing press — your code goes in, a website comes out."
iii.

Supabase

Where the data lives

A database, user logins, and file storage — without setting up servers. If your app needs to remember anything, this is where it remembers it.

"It's the filing cabinet — your app's memory, organized."
